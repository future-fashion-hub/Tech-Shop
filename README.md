# TechShop
**Архитектурный паттерн: MVC — проект организован по паттерну Model–View–Controller. Точка входа `src/index.ts` выполняет роль контроллера; модели находятся в `src/components/model`, представления — в `src/components/view`.**
## TechShop — небольшой фронтенд-магазин

Небольшой демонстрационный SPA-магазин на TypeScript + SCSS с простым клиентским управлением каталогом и корзиной. Проект показывает организацию компонентов, событийную архитектуру и сборку через Webpack.

**Кратко:** интерфейс: каталог карточек → открытие карточки в попапе → добавление в корзину → оформление заказа (локально, без реального платежа).

**Стек:**
- TypeScript
- SCSS (Sass)
- Webpack 5 (+ webpack-dev-server)
- PostCSS / Autoprefixer

**Структура (важные папки/файлы)**
- `src/` — исходники
    - `src/index.ts` — точка входа и настройка событий/взаимодействий
    - `src/pages/index.html` — HTML-шаблоны (template для карточки, попапов)
    - `src/components/` — UI-компоненты (Card, Page, Popup, Payment, Success и т.д.)
    - `src/components/model/` — модели данных (`CatalogModel`, `BusketModel`, `UserDataModel`)
    - `src/components/base/` — базовые утилиты/классы (EventEmitter, Component, AppApi)
    - `src/scss/` — стили проекта (переменные в `_variables.scss` и общие блоки)
- `public/images/` — статические изображения (логотип, товары и т.п.)

## Как запустить

Требования: Node.js (>=16), npm.

Установить зависимости и запустить dev-server:

```bash
npm install
npm start
```

После старта откроется dev-server (по умолчанию `http://localhost:8080`), страница будет автоматически пересобираться при правках.

Сборка production:

```bash
npm run build
```

Результат сборки — папка `dist/`.

## Архитектура и концепции

- Событийный брокер: `EventEmitter` используется как централизованный шина событий между моделями и представлениями (пример: `catalog:changed`, `card:open`, `order:change`).
- Модели: `CatalogModel`, `BusketModel`, `UserDataModel` содержат логику и состояние (каталог, корзина, данные заказа). Модели публикуют события при изменениях.
- Компоненты: базовый `Component<T>` реализует `render()` и работу с DOM-шаблонами; конкретные компоненты (`Card`, `Page`, `Popup`, `Payment`, `Busket`, `Success`) наследуются от него.
- Шаблоны: `src/pages/index.html` содержит `<template>`-блоки для карточек и модалок. Компоненты клонируют шаблон и заполняют данные.
- Стили: модульный SCSS + переменные в `src/scss/_variables.scss` для быстрой цветовой настройки.

## Где править данные и поведение
- Каталог / тестовые товары: `src/index.ts`.
- Компоненты: `src/components/view/` — визуальная логика.
- Модели: `src/components/model/` — бизнес-логика и состояние.
- Стили: `src/common.blocks/` и `src/scss/`.

## Активы и форматы
- Статические файлы (изображения, favicon и т.д.) кладутся в `public/` и доступны по `/images/...`.
- Предпочтительный формат изображений — PNG для растровых, SVG для векторных и иконок. В проекте присутствуют оба вида.

## Частые места для правок/настройки
- `src/index.ts` — логика загрузки и подмены данных
- `src/components/view/Payment.ts` — логика выбора способов оплаты
- `src/common.blocks/card.scss` — размеры и поведение карточек и изображений
